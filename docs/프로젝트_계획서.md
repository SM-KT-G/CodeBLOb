## 프로젝트의 목표
AI Hub의 K-Culture 관광 콘텐츠 특화 일본어 말뭉치(377,265개 문서)를 활용하여 일본인 관광객을 위한 한국 관광 안내 챗봇을 개발한다.  

### 핵심 목표
- **RAG(Retrieval Augmented Generation) 기반 정확한 답변 제공**
  - 일본어로 질문하면 관련 문서를 검색하여 AI가 자연스러운 답변 생성
  - 벡터 기반 시맨틱 검색으로 높은 검색 정확도 달성
  
- **외국인 관광객의 언어적 불편 해소**
  - 100% 일본어 인터페이스 제공
  - 한국 관광 정보에 특화된 도메인별 검색 (음식점, 역사, 레저, 자연, 쇼핑, 숙박)
  
- **확장 가능한 프로덕션급 아키텍처 구현**
  - FastAPI 기반 RESTful API 서버
  - PostgreSQL + pgvector를 활용한 벡터 DB
  - Docker 기반 컨테이너화로 배포 간소화
  - Redis 캐싱으로 응답 속도 최적화

- **최종 목표: 실제 서비스 가능한 MVP 완성**
  - 챗봇 프로토타입 완성
  - 한국 관광 산업에 실질적 기여 가능한 AI 응용 모델 제시
  - 다국어 확장 기반 마련 (영어, 중국어 등)

## 개발 내용

- **AI Hub K-Culture 일본어 데이터 활용**: 총 377,265개 문서 (음식, 역사, 레저, 자연, 쇼핑, 숙박 6개 도메인)
- **RAG 시스템 구축**: HuggingFace 임베딩 모델 + PostgreSQL pgvector 벡터 DB + OpenAI GPT-4
- **FastAPI 백엔드**: RESTful API, 비동기 처리, Redis 캐싱
- **Docker 컨테이너화**: 환경 독립적 배포 및 팀원 간 개발 환경 통일
- **웹 프론트엔드**: Streamlit/React 기반 일본어 채팅 UI (계획)

## 추진 전략

### 1주차: 프로젝트 기획 및 환경 구축
- **데이터 분석 및 기획**
  - AI Hub 데이터셋 분석 (377,265개 문서 구조 파악)
  - 6개 도메인 분류 및 데이터 품질 검증
  - RAG 시스템 아키텍처 설계 문서 작성
- **개발 환경 설정**
  - Python 3.10 가상환경 구축
  - Docker Compose 기반 인프라 구성 (PostgreSQL + Redis)
  - GitHub 프로젝트 저장소 생성 및 협업 규칙 수립

### 2주차: 데이터베이스 및 임베딩 시스템 구축
- **벡터 DB 구축**
  - PostgreSQL + pgvector 스키마 설계
  - 벡터 차원 최적화 (1024차원 결정)
  - 메타데이터 전략 수립 (저장 공간 80% 절감)
- **임베딩 파이프라인 개발**
  - HuggingFace 모델 선정 (multilingual-e5-large)
  - 임베딩 스크립트 작성 및 dry-run 테스트
  - GPU 최적화 적용 (M4 칩 활용)
- **대규모 임베딩 실행**
  - 377,265개 문서 임베딩 처리
  - 도메인별 데이터 검증

### 3주차: RAG 시스템 및 API 개발
- **RAG 컴포넌트 구현**
  - LangChain 기반 Retriever 개발
  - OpenAI GPT-4 LLM 통합
  - 일본어 프롬프트 템플릿 최적화
- **FastAPI 백엔드 개발**
  - RESTful API 설계 (/health, /rag/query)
  - 도메인별 필터링 기능 구현
  - Redis 캐싱 시스템 적용
- **성능 최적화**
  - 응답 속도 튜닝 (목표: 3초 이내)
  - 에러 핸들링 및 로깅 시스템 구축

### 4주차: 프론트엔드 개발 및 최종 통합
- **웹 UI 구현**
  - Streamlit/React 기반 일본어 채팅 인터페이스
  - 도메인 선택 및 출처 표시 기능
  - 사용자 경험 개선
- **통합 테스트 및 배포 준비**
  - 전체 시스템 통합 테스트
  - Docker 이미지 빌드 및 배포 자동화
  - API 문서 작성 (Swagger UI)
- **최종 점검 및 발표 준비**
  - 성능 평가 및 결과 분석
  - 발표 자료 제작 및 데모 시연 준비
  - 프로젝트 문서 최종 정리

### 핵심 전략
- **병렬 작업**: 백엔드/프론트엔드/인프라 역할 분담으로 효율 극대화
- **검증된 기술 스택**: LangChain, FastAPI, pgvector 등 검증된 도구 활용
- **비용 최적화**: 로컬 임베딩 모델 사용으로 비용 절감 ($1,131 → $0)
- **단계적 검증**: Dry-run 테스트 → 소규모 샘플 → 전체 적용으로 안정성 확보
